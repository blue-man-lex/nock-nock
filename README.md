![Foundry](https://img.shields.io/badge/Foundry%20VTT-11%2B-green.svg)

**Интерактивная система взлома замков для дверей и сундуков в Foundry VTT**

## 🎯 Обзор

Nock Nock - это модуль для Foundry VTT, который добавляет интерактивную систему взлома замков для дверей и сундуков. Модуль интегрируется с Treasure Hoard Manager и предоставляет игрокам увлекательный мини-игровой опыт взлома замков с различными механиками и настройками.

## ✨ Основные возможности

### 🔒 Типы замков
- **System Lock** - стандартные замки с настраиваемой сложностью
- **Formula Lock** - замки с динамической сложностью на основе формул
- **Key Only** - замки, которые можно открыть только с помощью ключа

### 🎲 Интерактивный взлом
- **Система бросков кубиков** - игроки делают проверку навыков для взлома
- **Выбор характеристики** - можно использовать различные характеристики (Ловкость, Интеллект и т.д.)
- **Преимущества/помехи** - поддержка механик advantage/disadvantage
- **Бонусы инструментов** - автоматическое обнаружение и применение бонусов от отмычек
- **Звуковые эффекты** - атмосферные звуки при успешном взломе

### 🗝️ Система ключей
- **Привязка ключей** - возможность привязать конкретный ключ к замку
- **Автообнаружение** - система автоматически проверяет наличие ключа у персонажа
- **Быстрое открытие** - кнопка использования ключа при его наличии

### 🎨 Премиальный интерфейс
- **BG3 стиль** - интерфейс в стиле Baldur's Gate 3
- **Адаптивный дизайн** - современный и интуитивный интерфейс
- **Анимации** - плавные переходы и визуальные эффекты

## 🚀 Установка

### Требования
- **Foundry VTT** версии 11 или выше (проверено на версии 13)
- **socketlib** - обязательная зависимость для мультиплеерных функций

### Установка через URL
1. Перейдите в **Administration → Modules**
2. Нажмите **Install Module**
3. Введите URL манифеста модуля
4. Установите и активируйте модуль

### Ручная установка
1. Скачайте модуль с GitHub
2. Распакуйте в папку `Data/modules/nock-nock`
3. Перезапустите Foundry VTT
4. Активируйте модуль в настройках

## 📖 Использование

### Для Мастера Игры (GM)

#### Настройка замков
1. Нажмите на иконку 🔒 в панели инструментов токенов
2. Выберите дверь или сундук для настройки
3. Укажите:
   - **Сложность замка** (DC)
   - **Тип замка** (System/Formula/Key Only)
   - **Ключ** (для типа Key Only)
   - **Дополнительные параметры**

#### Управление в игре
- **Двери** - клик правой кнопкой мыши по закрытой двери
- **Сундуки** - интеграция с Treasure Hoard Manager
- **Быстрая настройка** - через контекстное меню

### Для Игроков

#### Взлом замка
1. Попытайтесь открыть закрытый объект (дверь/сундук)
2. В интерфейсе взлома выберите:
   - **Характеристику** для проверки
   - **Преимущество/помеху** если применимо
   - **Дополнительный бонус** если нужно
3. Нажмите на кубик 🎲 для броска
4. При успехе замок откроется автоматически

#### Использование ключа
- Если у вас есть подходящий ключ, появится кнопка **"ИСПОЛЬЗОВАТЬ КЛЮЧ"**
- Ключ откроет замок без броска кубиков

### Интеграция с другими модулями

#### Treasure Hoard Manager
- Перехватывает открытие сундуков через хук `thm.preRenderInterface`
- Автоматически показывает интерфейс взлома для заблокированных контейнеров
- Использует флаги `treasure-hoard-manager.data.locked`

#### SocketLib
- Обеспечивает мультиплеерную синхронизацию
- Обработка запросов на взлом замков на стороне GM
- Безопасная передача данных между клиентами

#### libWrapper
- Перехватывает клики по дверям для интеграции
- Безопасная модификация стандартного поведения Foundry

### Система флагов

Модуль использует флаги для хранения данных о замках:

```javascript
// Пример флагов для замка
{
  "nock-nock": {
    "lockData": {
      "dc": 15,                    // Сложность замка
      "type": "system",            // Тип замка
      "key": {                     // Данные о ключе (опционально)
        "uuid": "Item.abc123",
        "name": "Золотой ключ"
      }
    }
  }
}
```

### События и хуки

Модуль использует следующие хуки Foundry:

- `init` - инициализация модуля
- `setup` - регистрация libWrapper
- `ready` - настройка сокетов
- `thm.preRenderInterface` - интеграция с THM
- `getSceneControlButtons` - добавление кнопки управления

## 🐛 Устранение неполадок

### Частые проблемы

**Q: Модуль не работает после установки**
A: Убедитесь, что socketlib установлен и активирован

**Q: Не открывается интерфейс взлома**
A: Проверьте, что у вас есть выбранный персонаж и вы не GM

**Q: Звуки не воспроизводятся**
A: Убедитесь, что файл `sounds/lock.wav` существует в папке модуля

**Q: Конфликт с другими модулями**
A: Проверьте консоль на ошибки и отключите конфликтующие модули
